CCNA Lab: Enable Password and Enable Secret Configuration

Topology
--------
R1 -------- SW1 -------- PC1
                     |-- PC2
                     |-- PC3

Interfaces
----------
R1 G0/0  <---->  SW1 G0/1
SW1 F0/1 <----> PC1
SW1 F0/2 <----> PC2
SW1 F0/3 <----> PC3

Objective
---------
1. Change hostnames of router and switch.
2. Configure an unencrypted enable password.
3. Verify password from user EXEC mode.
4. View password in running configuration.
5. Encrypt all current and future passwords.
6. Verify encrypted password.
7. Configure a more secure enable secret.
8. Test privileged EXEC access.
9. Identify encryption types.
10. Save configuration.

------------------------------------------------
Router Configuration (R1)
------------------------------------------------

Enable privileged mode
R1> enable

Enter global configuration mode
R1# configure terminal

Change hostname
Router(config)# hostname R1

Configure enable password (unencrypted)
R1(config)# enable password CCNA

Exit configuration mode
R1(config)# end

Test password
R1> enable
Password: CCNA

View running configuration
R1# show running-config

Encrypt all passwords
R1# configure terminal
R1(config)# service password-encryption
R1(config)# end

Verify encrypted password
R1# show running-config

Configure secure enable secret
R1# configure terminal
R1(config)# enable secret Cisco
R1(config)# end

Test privileged access
R1> enable
Password: Cisco

Verify encryption types
R1# show running-config

Expected Results
----------------
enable password  -> Type 7 encryption
enable secret    -> Type 5 encryption (MD5)

Save configuration
R1# copy running-config startup-config

Key Concepts
------------
enable password      : Legacy method, weak security
enable secret        : Strong encrypted password
service password-encryption : Encrypts plaintext passwords
Type 7 encryption    : Weak reversible encryption
Type 5 encryption    : MD5 hashed password
